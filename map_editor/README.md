# Редактор карты для fuel_game

Локальный веб-редактор: рисуете дороги, АЗС, НПЗ и точки светофоров прямо
на карте (та же основа OpenStreetMap, что использует сам фронтенд игры —
`frontend/src/pages/MapPage.tsx`), а редактор экспортирует всё в те же два
GeoJSON-файла и константы, которые читает бэкенд:

- `backend/data/mari_el_roads.geojson` — парсится
  `parse_road_features()` (`backend/app/services/map_import_service.py`).
- `backend/data/mari_el_stations.geojson` — парсится
  `parse_station_features()` (тот же файл).
- НПЗ — три константы `REFINERY_NAME` / `REFINERY_LATITUDE` /
  `REFINERY_LONGITUDE` в `backend/scripts/seed_game_data.py` (там нет
  отдельного GeoJSON).

Без сторонних зависимостей — сервер на стандартной библиотеке Python
(`http.server`), Leaflet вшит в `static/vendor/` (без обращения к CDN,
работает офлайн). Нужен только Python 3.9+.

## Запуск

```bash
cd map_editor
python3 server.py            # по умолчанию http://127.0.0.1:8765/
```

Откройте адрес в браузере.

## Как это устроено

**Режимы** (кнопки слева):

- **Дорога** — кликами добавляете точки ломаной, «Завершить дорогу»
  (Enter) открывает форму (тип: `local` / `trunk` / `primary`,
  макс. скорость, одностороннее движение). Пока рисуете, серыми точками
  подсвечиваются все уже существующие вершины дорог — щёлкнув рядом
  (в радиусе ~12 px), новая точка «прилипнет» к старой координате.
  Это важно: два перекрёстка должны иметь **точно одинаковые**
  координаты, иначе граф дорог в игре их не соединит.
- **АЗС** — клик ставит точку, форма: `osm_id` (уникальный, по нему
  идёт upsert при повторном импорте), название, базовая цена в рублях,
  населённый пункт.
- **НПЗ** — одна точка на весь проект; повторный клик спрашивает
  подтверждение на замену.
- **Светофор** — как и в самой игре, светофоры не задаются отдельно:
  бэкенд сам ставит их на каждом узле, где сходится **≥ 3** разных
  сегмента дороги (`MIN_INTERSECTION_DEGREE` в `map_import_service.py`).
  Точки, которые вы ставите в этом режиме — просто ориентир/проверка:
  клик снапится к ближайшей вершине дороги, а кнопка **«Проверить
  светофоры»** в шапке покажет для каждой точки, сколько сегментов
  сходится в этом узле и появится ли там реально светофор после
  импорта (зелёная точка на карте — ок, красная — нет).
- **Выбрать / удалить** — клик по объекту показывает и позволяет
  редактировать его свойства в правой панели, там же кнопка удаления.
  Тот же список объектов дублируется слева, с крестиком для удаления
  без выделения на карте.

**Проекты** — рабочее состояние (в рублях/градусах, без экспортных
преобразований) сохраняется на сервере в `projects/<имя>.json` кнопкой
«Сохранить». Выбор из выпадающего списка подгружает сохранённый проект.

**Экспорт**:

- «Скачать roads.geojson» / «Скачать stations.geojson» / «Скачать
  refinery snippet» — отдаёт файлы браузеру для скачивания.
- «Записать в output/ (сервер)» — то же самое, но сразу пишет в
  `map_editor/output/` на диске сервера (`mari_el_roads.geojson`,
  `mari_el_stations.geojson`, `refinery_seed_snippet.py`), плюс
  показывает предупреждения по светофорам, у которых меньше 3 сегментов.

Оба варианта экспорта сначала автоматически сохраняют текущее состояние
проекта на сервере, так что можно экспортировать сразу после рисования,
не нажимая «Сохранить» отдельно.

## Как перенести результат в fuel_game

1. В редакторе: «Записать в output/» (или скачайте оба файла).
2. Скопируйте экспортированные файлы поверх:
   ```bash
   cp map_editor/output/mari_el_roads.geojson backend/data/mari_el_roads.geojson
   cp map_editor/output/mari_el_stations.geojson backend/data/mari_el_stations.geojson
   ```
3. Если задавали НПЗ — откройте `map_editor/output/refinery_seed_snippet.py`
   и вручную перенесите три строки (`REFINERY_NAME`, `REFINERY_LATITUDE`,
   `REFINERY_LONGITUDE`) в `backend/scripts/seed_game_data.py`.
4. Граф дорог и светофоры пересчитываются на каждом старте бэкенда
   (`run_release_tasks` → `build_road_graph.main()`), так что просто
   передеплойте/перезапустите бэкенд.

## Технические детали снаппинга координат

Бэкенд дедуплицирует узлы дорог округлением координат до 6 знаков после
запятой (`_coordinate_key` в `map_import_service.py`, ~0.11 м). Нарисовать
мышью два конца дорог в абсолютно одинаковый пиксель нереально, поэтому
при экспорте (`editor/exporter.py::merge_road_vertices`) все вершины дорог
в проекте, находящиеся ближе `SNAP_TOLERANCE_M` (5 м) друг к другу,
объединяются в одну общую точку (среднее, округлённое до 6 знаков) — так
у пересечений получаются идентичные координаты, которые бэкенд опознает
как один узел. Проверка светофоров использует ту же логику склейки, чтобы
показывать реальную степень узла.

## Тесты

Без pytest — использован `unittest` из стандартной библиотеки:

```bash
cd map_editor
python3 -m unittest discover -s tests -t . -v
```

## Известные ограничения

- Геометрию существующей дороги нельзя перерисовать точечно — только
  удалить и нарисовать заново (свойства типа/скорости/одностороннего
  движения редактируются на месте).
- Экспорт `refinery_seed_snippet.py` — это фрагмент кода для ручного
  переноса в `seed_game_data.py`, редактор не патчит бэкенд-репозиторий
  автоматически.
- Требуется браузер с доступом к тайлам `tile.openstreetmap.org` для
  показа подложки карты; сам редактор (рисование/сохранение/экспорт)
  работает и без подложки, если тайлы недоступны.
